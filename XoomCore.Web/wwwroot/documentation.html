1. Introduction

Purpose

The purpose of this documentation is to provide a comprehensive overview of the XoomCore project, a .NET Core web project template designed for all developers. This document serves as a guide for developers, testers, and stakeholders involved in the project.

Scope

This documentation covers all aspects of the XoomCore project, including its architecture, features, API endpoints, testing procedures, deployment, and more. It is intended to assist team members in understanding the project and following best practices during development.

Document Conventions

Throughout this document:

  • Code snippets and examples are presented in a monospaced font.
  • Square brackets [ ] indicate optional items.
  • "Example" denotes an example scenario or placeholder text.

Intended Audience

This documentation is intended for developers, testers, project managers, and other stakeholders involved in the XoomCore project. A basic understanding of .NET Core, ASP.NET Core, Entity Framework Core, and RESTful API concepts is assumed.

2. Project Overview

XoomCore is a .NET Core web project template that provides developers with a solid foundation for building web applications. It follows the principles of a layered architecture, promoting separation of concerns and modularity. Developers can use this template as a starting point for various types of web projects.

3. Technologies Used

The XoomCore project template utilizes the following technologies and frameworks:

  • .NET Core 3.1 (or later)
  • ASP.NET Core 3.1 (or later)
  • Entity Framework Core
  • Serilog
  • Microsoft SQL Server (or another compatible database)

Layered Architecture:

  • Core Layer: Contains entities, interfaces for repositories and services, and business logic.
  • Application Layer: Contains view models, request/response models, services, and service interfaces.
  • Infrastructure Layer: Contains database context, repositories, and repository interfaces.
  • Web Layer: Entry point of the application, includes controllers and startup configuration.

6. Configuration

Database Connection

The XoomCore application uses Entity Framework Core to interact with the database. To configure the database connection, update the "DefaultConnection" value in the appsettings.json file with the appropriate connection string.


"ConnectionStrings": {
  "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=XoomCoreDb;Trusted_Connection=True;MultipleActiveResultSets=true"
}

Serilog Configuration

XoomCore uses Serilog for logging. The Serilog configuration is defined in the appsettings.json file. To customize the logging behavior, update the "Serilog" section in appsettings.json.


"Serilog": {
  "MinimumLevel": "Information",
  "WriteTo": [
    {
      "Name": "Console"
    }
  ]
}

8. Features

MenuCategory Entity CRUD Operations

The XoomCore application provides CRUD operations for managing MenuCategory entities. It allows users to create, read, update, and delete menu category records through a RESTful API.

Generic Repository Pattern

The application uses the generic repository pattern to abstract data access and provide a consistent interface for data operations.

9. API Endpoints

Get Menu Category List

URL: /api/menu-categories
Method: GET
Description: Retrieves a list of all menu categories.
Response: CommonResponse<IEnumerable<MenuCategoryResponse>>

Add Menu Category

URL: /api/menu-categories
Method: POST
Description: Adds a new menu category to the database.
Request Body: CreateMenuCategoryRequest
Response: CommonResponse<int> (The newly created menu category ID)
